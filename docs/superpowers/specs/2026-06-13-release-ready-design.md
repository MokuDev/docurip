# docurip Release-Ready Design

**Date:** 2026-06-13
**Version:** 0.2.0
**Status:** Approved

## Context

docurip is a Tauri v2 desktop app (Rust + React/TypeScript) for crawling documentation sites. Currently in development mode only — `bundle.active` is `false`, no updater, no installer. Goal: make it distributable as a Windows NSIS installer with auto-update via GitHub Releases.

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Platform | Windows only | Dev machine is win32, simplest path |
| Installer | NSIS (.exe) | Tauri v2 default, modern wizard, small footprint |
| Auto-Update | GitHub Releases + updater plugin | Zero infrastructure cost |
| CI/CD | None (local build) | User preference |
| Code Signing | None (for now) | Cost/effort; SmartScreen warning accepted |

## Architecture

### 1. Bundle Configuration (`src-tauri/tauri.conf.json`)

Enable bundling with NSIS target:

```json
{
  "bundle": {
    "active": true,
    "targets": ["nsis"],
    "icon": [
      "icons/32x32.png",
      "icons/128x128.png",
      "icons/128x128@2x.png",
      "icons/icon.icns",
      "icons/icon.ico"
    ],
    "windows": {
      "nsis": {
        "installMode": "both",
        "displayLanguageSelector": false
      }
    }
  }
}
```

- `installMode: "both"` — user can choose per-user (%LOCALAPPDATA%) or per-machine (%PROGRAMFILES%)
- Icons already exist in `src-tauri/icons/` (auto-generated by Tauri)

### 2. Updater Plugin

**Dependencies to add:**
- `Cargo.toml`: `tauri-plugin-updater = "2"`
- `package.json`: `@tauri-apps/plugin-updater = "^2.0.0"`

**Registration (`lib.rs`):**
```rust
.plugin(tauri_plugin_updater::Builder::new().build())
```

**Permissions (`capabilities/default.json`):**
```json
"updater:default"
```

**Config (`tauri.conf.json`):**
```json
{
  "plugins": {
    "updater": {
      "endpoints": [
        "https://github.com/{owner}/{repo}/releases/latest/download/latest.json"
      ],
      "pubkey": "<generated-by-tauri-signer>"
    }
  }
}
```

The `pubkey` is generated once with `npx tauri signer generate` and the private key is kept secret (used during build to sign updates).

**Frontend update check:**
Add a check in `App.tsx` or a dedicated hook that calls the updater API on app start and shows a notification dialog when an update is available.

### 3. NSIS Installer Behavior

| Aspect | Behavior |
|--------|----------|
| Install location | User-selectable (per-user or per-machine) |
| Shortcuts | Desktop + Start Menu (default) |
| Uninstall | Clean removal via Windows Add/Remove Programs |
| License | Show LICENSE file if present |
| Upgrade | In-place upgrade, preserves user data (%APPDATA%) |

### 4. Metadata Cleanup

| File | Field | Current | New |
|------|-------|---------|-----|
| `Cargo.toml` | `authors` | `["you"]` | `["Docurip Contributors"]` |
| `Cargo.toml` | `description` | already set | keep |
| Versions | all three files | `0.2.0` | keep in sync |

### 5. Release Script (`scripts/release.ps1`)

PowerShell script that:
1. Validates version consistency (Cargo.toml = package.json = tauri.conf.json)
2. Runs `npm run tauri build`
3. Reports output path of NSIS installer
4. Optionally creates GitHub Release with `gh release create v{version}` and uploads the .exe

### 6. Build Output

```
src-tauri/target/release/bundle/nsis/
  docurip_0.2.0_x64-setup.exe       ← Distributable installer
```

### 7. Update Signing Flow

1. Generate keypair once: `npx tauri signer generate -w ~/.tauri/docurip.key`
2. Public key goes into `tauri.conf.json` → `plugins.updater.pubkey`
3. Private key used at build time (env var `TAURI_SIGNING_PRIVATE_KEY`)
4. Build produces `.exe.sig` signature file alongside installer
5. Upload both `.exe` and `.exe.sig` to GitHub Release
6. `latest.json` manifest auto-generated by Tauri build

## Files to Modify

| File | Change |
|------|--------|
| `src-tauri/tauri.conf.json` | Enable bundle, add NSIS config, add updater config |
| `src-tauri/Cargo.toml` | Add `tauri-plugin-updater` dep, fix authors |
| `package.json` | Add `@tauri-apps/plugin-updater` dep, add `release` script |
| `src-tauri/src/lib.rs` | Register updater plugin |
| `src-tauri/capabilities/default.json` | Add updater permission |
| `src/App.tsx` (or new hook) | Add update check on startup |
| `scripts/release.ps1` | New — local release script |

## Out of Scope

- Code signing (future: EV certificate)
- CI/CD (future: GitHub Actions)
- macOS/Linux builds
- Custom NSIS branding images (header/sidebar)
- Download assets feature (existing, unrelated to packaging)

## Success Criteria

1. `npm run tauri build` produces a working `.exe` installer in `src-tauri/target/release/bundle/nsis/`
2. Installer installs docurip, creates shortcuts, registers uninstaller
3. App starts correctly from installed location
4. Update check runs on startup (no updates available yet, but no errors)
5. `scripts/release.ps1` builds and optionally publishes to GitHub Releases
